## What is InterSystems IRIS?

InterSystems IRIS Data Platform is a high-performance, multi-model data platform that combines a robust database engine, an interoperability framework, advanced analytics capabilities, and an API management layer within a single unified product.

Unlike conventional database systems that force you to choose between relational, document, or key-value paradigms, IRIS seamlessly supports **multiple data models simultaneously**:

- **Relational SQL:** Query your data using standard SQL syntax.
- **Object-Oriented:** Map classes directly to persistent storage with no ORM overhead.
- **Document JSON:** Store and query semi-structured JSON documents natively.
- **Key-Value:** Ultra-fast direct access using global data structures.

This multi-model approach eliminates the need for complex data transformation layers and dramatically simplifies your application architecture.

### Key Features of IRIS

1. **Multi-Model Database Engine** works with SQL, objects, documents, and key-value stores simultaneously without data duplication.
2. **Built-in Interoperability** connects disparate systems using HL7, FHIR, REST, SOAP, and custom protocols right out of the box.
3. **Real-Time Analytics** runs analytical queries directly against transactional data with no ETL pipeline required.
4. **Horizontal Scalability** scales seamlessly with Enterprise Cache Protocol and sharding capabilities.
5. **HIPAA and Regulatory Compliance** provides enterprise-grade security, encryption, and auditing features baked into the platform.

## What is ObjectScript?

ObjectScript is the native programming language of the InterSystems IRIS platform. It is a powerful, dynamically-typed, object-oriented language specifically designed for building data-intensive applications with direct access to the underlying database engine.

<blockquote>ObjectScript gives you the unique ability to write application logic that lives alongside your data, eliminating the impedance mismatch between code and storage.</blockquote>

### ObjectScript Syntax Fundamentals

If you are coming from languages like Python, JavaScript, or Java, ObjectScript syntax will feel different at first, but it is remarkably expressive once you get the hang of it. Let us explore the core building blocks.

#### Variables and Data Types

ObjectScript uses a loosely typed variable system. Variables do not require explicit type declarations:

```sql title="basics.cls"
 // Setting variables
 Set name = "John Doe"
 Set age = 30
 Set isActive = 1

 // String concatenation using the underscore operator
 Set fullGreeting = "Hello, " _ name _ "! You are " _ age _ " years old."
 Write fullGreeting, !
```

Notice that ObjectScript uses the underscore **_** operator for string concatenation, and the exclamation mark **!** represents a newline in output statements.

#### Control Flow

ObjectScript supports traditional control flow constructs with its own syntax flavor:

```sql title="controlflow.cls"
 // If-Else statement
 If (age >= 18) {
     Write "You are an adult.", !
 } ElseIf (age >= 13) {
     Write "You are a teenager.", !
 } Else {
     Write "You are a child.", !
 }

 // For loop using start:step:end pattern
 For i = 1:1:10 {
     Write "Iteration: " _ i, !
 }

 // While loop
 Set counter = 0
 While (counter < 5) {
     Set counter = counter + 1
     Write "Count: " _ counter, !
 }
```

The **For** loop syntax uses a **start:step:end** pattern, which is concise and powerful for numeric iterations.

### Working with Classes

One of the most compelling features of ObjectScript is its deep integration with an object-oriented class system that maps directly to persistent storage. Here is how you define a basic persistent class:

```sql title="Patient.cls"
Class Healthcare.Patient Extends %Persistent
{

Property Name As %String(MAXLEN = 100) [ Required ];

Property DateOfBirth As %Date;

Property Email As %String(MAXLEN = 200);

Property Status As %String [ InitialExpression = "Active" ];

Index NameIndex On Name;

Index EmailIndex On Email [ Unique ];

/// Create a new patient record
ClassMethod CreatePatient(
    name As %String,
    dob As %Date,
    email As %String
  ) As %Status
{
    Set patient = ..%New()
    Set patient.Name = name
    Set patient.DateOfBirth = dob
    Set patient.Email = email

    Set sc = patient.%Save()
    If $$$ISERR(sc) {
        Write "Error saving patient: "
            _ $System.Status.GetErrorText(sc), !
    } Else {
        Write "Patient created with ID: "
            _ patient.%Id(), !
    }

    Quit sc
}

/// Find a patient by name using Dynamic SQL
ClassMethod FindByName(
    name As %String
  ) As Healthcare.Patient
{
    Set query = "SELECT ID FROM Healthcare.Patient WHERE Name = ?"
    Set stmt = ##class(%SQL.Statement).%New()
    Set sc = stmt.%Prepare(query)
    Set rs = stmt.%Execute(name)

    If rs.%Next() {
        Quit ..%OpenId(rs.%Get("ID"))
    }

    Quit ""
}

}
```

There are several groundbreaking concepts embedded in this class definition:

1. **%Persistent inheritance**: By extending **%Persistent**, this class automatically gains full database persistence. No ORM configuration, no migration files, no separate schema definitions.
2. **Property constraints**: Properties support inline validation rules like **MAXLEN** and **Required** directly in the class definition.
3. **Automatic SQL projection**: This class is instantly queryable via SQL as the table **Healthcare.Patient** without any additional setup.
4. **Index definitions**: Database indexes are declared right alongside the properties they optimize.

### Globals: The Secret Weapon

Under the hood, all data in InterSystems IRIS is stored in structures called **Globals**. Globals are sparse, hierarchical arrays that provide incredibly fast key-value access. Understanding globals gives you superpowers when working with IRIS:

```sql title="globals.cls"
 // Setting global values
 Set ^Inventory("Laptop", "Dell") = 150
 Set ^Inventory("Laptop", "HP") = 200
 Set ^Inventory("Phone", "iPhone") = 500
 Set ^Inventory("Phone", "Samsung") = 350

 // Reading a global value
 Write ^Inventory("Laptop", "Dell"), !

 // Iterating through globals using $Order
 Set category = ""
 For {
     Set category = $Order(^Inventory(category))
     Quit:category=""

     Set brand = ""
     For {
         Set brand = $Order(^Inventory(category, brand))
         Quit:brand=""

         Write category _ " - " _ brand _ ": "
             _ ^Inventory(category, brand), !
     }
 }
```

Globals prefixed with the caret **^** symbol are stored persistently in the database. They are extraordinarily performant because they bypass the SQL parsing layer entirely, offering direct B-tree access to the underlying storage engine.

### SQL Integration

One of the most elegant aspects of IRIS is how seamlessly ObjectScript integrates with SQL. You can embed SQL queries directly inside your ObjectScript methods using both Embedded SQL and Dynamic SQL:

```sql title="queries.cls"
ClassMethod GetActivePatients() As %Status
{
    // Dynamic SQL approach
    Set query = "SELECT Name, Email, Status "
        _ "FROM Healthcare.Patient "
        _ "WHERE Status = 'Active' "
        _ "ORDER BY Name"
    Set stmt = ##class(%SQL.Statement).%New()
    Set sc = stmt.%Prepare(query)
    Set rs = stmt.%Execute()

    While rs.%Next() {
        Write "Name: " _ rs.%Get("Name"), !
        Write "Email: " _ rs.%Get("Email"), !
        Write "Status: " _ rs.%Get("Status"), !
        Write "---", !
    }

    Quit $$$OK
}

ClassMethod CountPatientsByStatus() As %Status
{
    Set query = "SELECT Status, COUNT(*) AS Total "
        _ "FROM Healthcare.Patient "
        _ "GROUP BY Status"
    Set stmt = ##class(%SQL.Statement).%New()
    Set sc = stmt.%Prepare(query)

    If $$$ISERR(sc) {
        Quit sc
    }

    Set rs = stmt.%Execute()

    While rs.%Next() {
        Write rs.%Get("Status") _ ": "
            _ rs.%Get("Total") _ " patients", !
    }

    Quit $$$OK
}
```

IRIS supports two SQL paradigms:

- **Embedded SQL**: SQL statements are compiled directly into the ObjectScript routine at build time, offering maximum performance.
- **Dynamic SQL**: SQL statements are prepared and executed at runtime, providing flexibility for dynamic query construction.

### REST API Development

Building RESTful APIs with IRIS is remarkably straightforward. Here is how you set up a REST dispatch class:

```sql title="PatientAPI.cls"
Class Healthcare.API.PatientREST Extends %CSP.REST
{

Parameter UseSession = 1;

// URL routing is defined in the XData UrlMap block
// using XML format with Routes and Route elements.
// Routes map HTTP methods to ClassMethods:
//   GET /patients -> GetAllPatients
//   GET /patients/:id -> GetPatient
//   POST /patients -> CreatePatient

ClassMethod GetAllPatients() As %Status
{
    Set array = []
    Set query = "SELECT ID, Name, Email, Status "
        _ "FROM Healthcare.Patient"
    Set stmt = ##class(%SQL.Statement).%New()
    Do stmt.%Prepare(query)
    Set rs = stmt.%Execute()

    While rs.%Next() {
        Set obj = {}
        Set obj.id = rs.%Get("ID")
        Set obj.name = rs.%Get("Name")
        Set obj.email = rs.%Get("Email")
        Set obj.status = rs.%Get("Status")
        Do array.%Push(obj)
    }

    Write array.%ToJSON()
    Quit $$$OK
}

ClassMethod GetPatient(id As %String) As %Status
{
    Set patient = ##class(Healthcare.Patient).%OpenId(id)

    If patient = "" {
        Set %response.Status = "404 Not Found"
        Write {"error": "Patient not found"}.%ToJSON()
        Quit $$$OK
    }

    Set obj = {}
    Set obj.id = patient.%Id()
    Set obj.name = patient.Name
    Set obj.email = patient.Email
    Set obj.status = patient.Status

    Write obj.%ToJSON()
    Quit $$$OK
}

ClassMethod CreatePatient() As %Status
{
    Set body = {}.%FromJSON(%request.Content)

    Set sc = ##class(Healthcare.Patient).CreatePatient(
        body.name,
        body.dob,
        body.email
    )

    If $$$ISOK(sc) {
        Set %response.Status = "201 Created"
        Write {"message": "Patient created"}.%ToJSON()
    } Else {
        Set %response.Status = "400 Bad Request"
        Write {"error": "Failed to create"}.%ToJSON()
    }

    Quit $$$OK
}

}
```

This single class gives you a fully functional REST API with routing, JSON serialization, error handling, and HTTP status code management, all tightly integrated with your data model.

## IRIS vs Traditional Databases

Understanding how IRIS compares to conventional database systems helps clarify where it truly excels:

- **Data Model**: Traditional RDBMS uses a single relational model, while IRIS supports multi-model access including SQL, Object, Document, and Key-Value simultaneously.
- **Object Persistence**: Traditional systems require an ORM layer such as Hibernate. IRIS provides native object-relational mapping with zero configuration.
- **Interoperability**: Traditional databases need external middleware. IRIS ships with a built-in integration engine.
- **Analytics**: Traditional setups demand a separate OLAP system. IRIS unifies OLTP and OLAP in one platform.
- **API Development**: Traditional platforms rely on external frameworks. IRIS includes built-in REST and SOAP support.
- **Healthcare Standards**: Traditional databases depend on third-party libraries. IRIS natively supports HL7, FHIR, and CDA standards.

## Best Practices for ObjectScript Development

To write production-quality ObjectScript code, keep these principles in mind:

### 1. Use Proper Error Handling

Always check status codes returned by system methods:

```sql title="errorhandling.cls"
ClassMethod SafeOperation() As %Status
{
    Set sc = $$$OK

    Try {
        Set patient = ##class(Healthcare.Patient).%New()
        Set patient.Name = "Jane Smith"
        Set sc = patient.%Save()

        If $$$ISERR(sc) {
            Throw ##class(%Exception.StatusException)
                .CreateFromStatus(sc)
        }

        Write "Operation completed successfully.", !
    } Catch ex {
        Set sc = ex.AsStatus()
        Write "Error: " _ ex.DisplayString(), !
    }

    Quit sc
}
```

### 2. Leverage Macros for Clean Code

InterSystems provides powerful macro definitions that streamline common patterns:

```sql title="macros.cls"
 // Instead of manual status checking
 Set sc = patient.%Save()
 If $$$ISERR(sc) {
     // handle error
 }

 // Use the ThrowOnError macro for cleaner code
 $$$ThrowOnError(patient.%Save())
```

### 3. Use %UnitTest for Testing

IRIS includes a built-in unit testing framework:

```sql title="TestPatient.cls"
Class Healthcare.Tests.TestPatient Extends %UnitTest.TestCase
{

Method TestCreatePatient()
{
    Set sc = ##class(Healthcare.Patient).CreatePatient(
        "Test User",
        $ZDateH("2000-01-15", 3),
        "test@example.com"
    )

    Do $$$AssertStatusOK(sc, "Patient creation should succeed")
}

Method TestFindByName()
{
    Set patient = ##class(Healthcare.Patient)
        .FindByName("Test User")
    Do $$$AssertNotEquals(patient, "", "Patient should be found")
    Do $$$AssertEquals(patient.Name, "Test User", "Name should match")
}

}
```
